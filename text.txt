## 1. What is HTML?

HTML (HyperText Markup Language) is used to **create the structure and content of web pages**. Everything you see on a webpage‚Äîtext, headings, images, links‚Äîcomes from HTML.

> **Remember:** HTML is the skeleton of a website.

---

## 2. How HTML Works

HTML uses **elements**.

An element has:

* **Opening tag** ‚Üí `<p>`
* **Content** ‚Üí `Hello`
* **Closing tag** ‚Üí `</p>`

Example:

```html
<p>Hello</p>
```

---

## 3. Void Elements

Some elements **do not have closing tags and cannot contain content**. These are called **void elements**.

Examples:

```html
<img>
<br>
<hr>
<input>
```

You may also see:

```html
<img />
```

Both are valid in HTML5.

> **Key rule:** Void elements only have a start tag.

---

## 4. What are Attributes?

An **attribute** gives extra information about an element and controls **how it behaves or looks**.

**Syntax:**

```html
<element attribute="value"></element>
```

> **Easy idea:** Element = what it is, Attribute = how it works

---

## 5. Common Attributes

### a) Links ‚Äì `href` and `target`

```html
<a href="https://example.com" target="_blank">Visit</a>
```

* `href` ‚Üí where the link goes (required)
* `target="_blank"` ‚Üí opens in new tab

---

### b) Images ‚Äì `src` and `alt`

```html
<img src="cat.jpg" alt="A sleeping cat">
```

* `src` ‚Üí image location (required)
* `alt` ‚Üí text description (for accessibility)

---

## 6. Boolean Attributes

Boolean attributes **do not need values**.

If present ‚Üí **true**
If absent ‚Üí **false**

Examples:

```html
<input type="checkbox" checked>
<input type="text" disabled>
<input type="text" required>
```

Common boolean attributes:

* `checked`
* `disabled`
* `readonly`
* `required`

---

## 7. Input and the `type` Attribute

The `type` attribute tells the browser what kind of input it is.

```html
<input type="text">
<input type="checkbox">
<input type="email">
```

---

## 8. HTML, CSS & JavaScript ‚Äì The Big Picture

* **HTML** ‚Üí Structure & content
* **CSS** ‚Üí Design & styling
* **JavaScript** ‚Üí Interactivity & logic

üè† **House Analogy**

* HTML = bricks & walls
* CSS = paint & decoration
* JavaScript = electricity & water

---

## 9. Accessibility (Important)

Accessibility means making websites usable for **everyone**, including people with disabilities.

Examples:

* `alt` text for images
* `required` for forms
* `readonly` for inputs

---
